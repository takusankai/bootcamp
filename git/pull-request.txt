[github の pull request (プルリクエスト) を理解する]

pull requestとは、オンライン上のレポジトリを更新した際に、他にもレポジトリ編集者がいる場合に、その編集内容を受け取って(pull)くださいというリクエストを出すこと。
他の開発者の方に見て貰って（レビュー）問題が無ければマージする。

github上で緑色のCompare & pull requestボタンをクリックすることで作成できる。
他にもCLIやアプリケーションなどから作る方法が公式ドキュメントに紹介されている。
タイトルとコメントを適切に行う。内容、レビューしてもらいたいこと、備考などを項目建てて書くと良い。

大切なのは、どこを何故直したのか？のような送った側の意図をレビューする側が理解できることである。
またTRYOUTスプレッドシートにもあるが、差分の粒度にも気を配る。趣旨を決め、それに逸れた変更は含めない。
